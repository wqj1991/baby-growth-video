> **设计师**: UI Designer (像素君)

> **版本**: v1.0

## 一、项目现状分析

### 技术栈
- **框架**: React 18 + TypeScript + Vite
- **桌面端**: Tauri 2.0
- **样式**: Tailwind CSS + PostCSS
- **路由**: React Router v6
- **状态管理**: Zustand
- **图标**: Lucide React

### 当前架构
- 侧边栏导航 (Layout) → 首页、宝宝信息、视频制作、历史记录
- 新建项目四步向导 (CreateProjectPage)
- 项目管理三子页：概览、阶段选择、视频生成、历史记录
- 基础样式系统已有 Tailwind 基础

### 设计痛点
1. 当前侧边栏 Logo 区和宝宝信息耦合度高，缺少清晰分隔
2. 首页缺乏情感化设计和行动引导
3. 步骤指示器视觉一致性不足
4. 照片选择器的空状态和操作指引不够清晰
5. 整体色彩偏冷，缺少"成长""温暖"的情感表达

## 二、全新设计系统

### 2.1 品牌色彩

| 类别 | 色值 | 用途 |
|------|------|------|
| 品牌主色 | `#534AB7` (Primary 600) | 导航高亮、主要按钮、选中态 |
| 品牌浅色 | `#EEEDFE` (Primary 50) | 背景填充、活跃状态 |
| 辅助品牌 | `#7F77DD` (Primary 400) | 次要强调、描述文字 |
| 成功色 | `#639922` (Green 400) | 完成状态、正向反馈 |
| 警告色 | `#BA7517` (Amber 400) | 待处理阶段、提示 |
| 错误色 | `#E24B4A` (Red 400) | 删除、失败、异常 |
| 信息色 | `#378ADD` (Blue 400) | 提示框、帮助说明 |

### 2.2 字体系统

| 层级 | 字号 | 字重 | 行高 | 颜色 |
|------|------|------|------|------|
| 标题 (h1) | 18px | 500 | 1.4 | `#2C2C2A` (Gray 900) |
| 副标题 (h2) | 15px | 500 | 1.4 | `#444441` (Gray 800) |
| 正文 | 13px | 400 | 1.6 | `#5F5E5A` (Gray 600) |
| 提示文字 | 12px | 400 | 1.5 | `#888780` (Gray 400) |
| 辅助文字 | 11px | 400 | 1.5 | `#B4B2A9` (Gray 200) |

### 2.3 间距系统 (8px 基准)

- `4px` — 极小间距（图标与文字间隔）
- `8px` — 小间距（标签内元素）
- `12px` — 中等间距（卡片内部）
- `16px` — 标准间距（区块间距）
- `24px` — 大间距（页面区块）
- `32px` — 超大间距（页面外边距）

### 2.4 圆角系统

- `6px` — 小按钮、标签
- `8px` — 交互元素、输入框
- `12px` — 卡片、容器
- `16px` — 大型模态框

## 三、关键页面设计方案

### 3.1 首页改版

**设计目标**: 营造温暖的氛围，引导用户快速行动

#### 改动要点
1. **新增顶部状态区** — 显示当前宝宝信息和最近操作，建立情感连接
2. **双列操作按钮** — "新建项目"（品牌色高亮）和"继续编辑"（白色边框）明确区分主次行动
3. **项目卡片增强** — 增加进度条、状态标签、创建时间
4. **空状态优化** — 使用友好的引导文案和图标

#### 组件结构
```
首页
├── 顶部状态栏（宝宝头像 + 昵称 + 最近操作）
├── 双列行动区（新建 / 继续编辑）
├── 最近项目列表（卡片 + 进度条）
└── 空状态引导（友好提示）
```

### 3.2 侧边栏导航优化

**设计目标**: 清晰的层级、明确的状态反馈

#### 改动要点
1. **Logo 区域精简** — 去除冗余的文字描述，强化品牌识别
2. **宝宝信息分离** — 独立卡片区域，与导航分离
3. **导航层级扁平** — 减少嵌套，最多两级
4. **活跃状态明确** — 使用品牌色背景和边框双重标识
5. **底部设置固定** — 始终保持在侧边栏底部

#### 导航项顺序
1. 首页（默认）
2. 视频制作（进入项目管理）
3. 项目列表（多项目管理）
4. 历史记录（全局历史）
5. 设置（底部固定）

### 3.3 视频制作流程页

**设计目标**: 降低认知负荷，清晰展示进度

#### 步骤指示器设计
- **已完成** — 绿色圆形 + 对勾图标
- **进行中** — 品牌色圆形 + 数字编号 + 外部光晕环
- **未完成** — 灰色空心圆形 + 灰色数字
- **连接线** — 已完成段为实线，未开始段为虚线

#### 每步内容卡片
- 顶部：步骤标题 + 统计信息（如"共找到 128 张照片"）
- 中部：操作区域（上传、选择、配置等）
- 底部：分页控制（上一步 / 下一步 / 确认）

### 3.4 照片选择器优化

#### 网格布局
- 移动端: `grid-cols-3` (150px)
- 平板: `grid-cols-4` (180px)
- 桌面: `grid-cols-5` (200px)

#### 卡片设计
- 默认态: 白色背景 + 灰色边框
- 悬停态: 缩放 1.02 + 品牌色边框
- 选中态: 蓝色边框 + 右下角选中徽章
- 已锁定: 绿色边框 + 金色描边徽章

#### 操作反馈
- 批量选择: 顶部显示 "已选择 12/48 张"
- 取消选择: 点击已选中卡片移除
- 全部选中: 右上角全选复选框

## 五、交互与动效规范

### 5.1 过渡动画
- 标准过渡: `150ms ease` (按钮悬停、焦点切换)
- 中等过渡: `300ms ease` (面板展开收起)
- 长过渡: `500ms ease` (页面切换)

### 5.2 微交互
- 按钮点击: `scale(0.98)` 微缩效果
- 卡片悬停: `translateY(-2px)` 轻微上浮
- 进度条: 平滑宽度变化 (`duration-300`)

### 5.3 加载状态
- 骨架屏: 用于照片网格加载
- 进度指示: 环形进度条用于视频生成过程
- 步骤加载: 逐步高亮动画

## 六、无障碍设计

| 标准 | 实现方式 |
|------|----------|
| 色彩对比 | 正文 ≥ 4.5:1，大字 ≥ 3:1 |
| 键盘导航 | 所有交互元素支持 Tab 键聚焦 |
| 焦点指示 | 2px 品牌色外轮廓 + 2px 偏移 |
| 屏幕阅读器 | 语义化 HTML + ARIA 标签 |
| 触摸目标 | 最小 44×44px 交互区域 |
| 减少动效 | 尊重系统 `prefers-reduced-motion` |
