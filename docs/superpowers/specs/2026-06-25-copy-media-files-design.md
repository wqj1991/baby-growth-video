# 媒体文件复制到项目目录设计

## 问题描述

前端使用 `file://` 协议直接访问原始媒体文件路径时，遇到跨域问题（CORS），导致图片无法正常加载。

## 解决方案

扫描媒体文件时，将识别到的照片和视频复制到应用数据目录下的项目专属文件夹中，数据库中存储复制后的路径。

## 目录结构

```
{数据目录}/baby-growth-video/
├── app.db                    # 数据库文件
├── projects/                 # 项目媒体文件目录
│   ├── {project_id}/         # 项目ID目录
│   │   ├── photos/           # 照片目录
│   │   │   ├── {uuid}_{original_name}.jpg
│   │   │   └── ...
│   │   └── videos/           # 视频目录
│   │       ├── {uuid}_{original_name}.mp4
│   │       └── ...
└── frames/                   # 视频截图目录（已有）
    └── {video_id}/
```

## 修改内容

### 1. 后端修改

#### 1.1 media.rs
- 添加项目目录创建函数 `get_project_media_dir(project_id)`
- 修改 `scan_media_folder` 函数：
  - 创建项目照片/视频目录
  - 复制文件到项目目录
  - 数据库中存储复制后的路径

#### 1.2 db.rs
- 修改 `delete_project` 函数：删除项目时同时删除项目目录下的媒体文件

### 2. 前端修改

#### 2.1 PeriodSelectPage.tsx
- 图片显示路径无需修改（数据库已存储复制后的路径）

### 3. 文件命名规则

复制后的文件命名：`{UUID}_{原始文件名}`

- UUID 防止文件名冲突
- 保留原始文件名便于识别

## 实现步骤

1. 添加项目目录管理函数
2. 修改扫描逻辑，复制文件并存储新路径
3. 修改删除项目逻辑，清理项目目录
4. 更新数据库中 Photo/Video 的 file_path 字段
5. 验证构建和功能

## 注意事项

- 复制过程中需要处理文件重名情况（使用 UUID 前缀）
- 删除项目时需要递归删除目录
- 需要处理复制失败的异常情况
- 大文件复制可能需要较长时间，建议在后台执行
