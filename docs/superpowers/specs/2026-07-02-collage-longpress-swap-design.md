# 拼图长按换位交互设计

**日期**: 2026-07-02  
**主题**: 拼图功能实现长按区分拖拽（短按格内平移 / 长按块间换位）

## 1. 背景

当前拼图工作区同时存在两类交互：
- 画布区域内图片平移（用于调整构图）
- 画布区域间位置交换（用于调整拼图顺序）

现状存在空格键模式切换与区域 HTML5 拖拽并行，心智负担偏高，且短按与长按意图未被明确区分。

## 2. 目标

实现统一手势交互，满足以下规则：
1. 短按直接拖动（默认行为）= 单图局部平移。
2. 平移范围锁定在当前格子容器内，超出自动回弹（本质为边界约束），不换格。
3. 长按 0.3s 以上再拖动 = 互换拖拽模式。
4. 长按触发后出现明显视觉变化（块放大、虚线边框、半透明阴影）。
5. 换位模式下拖动到另一拼图块附近松手，触发吸附后互换。
6. 拖到空白区域或未命中目标时回弹，不改变顺序。

## 3. 已确认参数

1. 长按阈值：300ms。
2. 长按防误触位移阈值：4px。
3. 吸附判定方式：最近目标块距离判定。
4. 吸附距离阈值：24px。

## 4. 设计范围

### 4.1 In Scope

- 改造拼图画布区手势引擎为 Pointer Events 统一处理。
- 替换画布区域 HTML5 `draggable` 换位路径。
- 保留并复用现有格内平移边界计算逻辑。
- 增加换位模式视觉态与吸附高亮反馈。

### 4.2 Out of Scope

- 不调整右侧列表拖拽排序能力。
- 不新增后端接口或数据库字段。
- 不改动拼图导出参数与生成逻辑。

## 5. 方案对比与结论

### 5.1 方案 A（选定）

使用 Pointer 事件 + 手势状态机统一处理短按平移与长按换位。

**优点**：
- 逻辑集中，阈值判定一致。
- 鼠标与触控输入路径统一，扩展性强。
- 能平滑替代空格键模式切换。

**代价**：
- 需要中等范围前端交互重构。

### 5.2 未选方案

- 方案 B：保留 HTML5 `draggable` 并加长按门控。风险是事件冲突与跨端一致性较弱。
- 方案 C：浮层代理块换位。体验更炫但复杂度偏高，不适合本次迭代。

## 6. 交互架构

采用单指针状态机：
- `idle`
- `pressing`
- `panning`
- `swapping`
- `settling`

状态转移：
1. `pointerdown` -> `pressing`，记录起点与时间戳，启动 300ms 计时。
2. `pressing` 期间位移 > 4px -> `panning`（取消长按计时）。
3. `pressing` 达到 300ms 且位移 <= 4px -> `swapping`。
4. `swapping` 松手且命中目标 -> 交换位置；未命中 -> 回弹。
5. 收尾动画后回到 `idle`。

## 7. 组件与状态设计

实施位置：`src/components/CollageWorkspace.tsx`

### 7.1 新状态（组件内）

`gestureState`：
- `phase: 'idle' | 'pressing' | 'panning' | 'swapping' | 'settling'`
- `pointerId: number | null`
- `sourceRegionIdx: number | null`
- `startClientX: number`
- `startClientY: number`
- `lastClientX: number`
- `lastClientY: number`
- `pressStartAt: number`
- `swapCandidateRegionIdx: number | null`

Refs：
- `longPressTimerRef`：300ms 计时器。
- `regionRectsRef`：区域矩形缓存。
- `gestureRuntimeRef`：减少频繁 setState 导致的抖动（可选实现）。

### 7.2 事件入口

区域容器统一改为：
- `onPointerDown`
- `onPointerMove`
- `onPointerUp`
- `onPointerCancel`

关键点：
- `pointerdown` 后执行 `setPointerCapture(pointerId)`。
- 仅处理当前激活 `pointerId`。
- 画布区移除 `draggable/onDragStart/onDrop` 换位路径。

## 8. 详细交互流程

### 8.1 短按平移（默认）

1. 用户按下并立即拖动，若位移超过 4px：
   - 进入 `panning`。
   - 调用现有边界计算进行 offset clamp。
2. 平移过程中只改变当前格子内图片偏移，不改变格子映射。
3. 松手后保留偏移结果。

### 8.2 长按换位

1. 用户按下后保持位移 <= 4px。
2. 300ms 到达后进入 `swapping`。
3. 显示换位视觉态：
   - 当前块 scale 放大（建议 1.03~1.06）
   - 虚线边框
   - 半透明阴影
4. 跟随指针移动，并计算最近目标块中心距离。
5. 若最近距离 <= 24px：
   - 激活 `swapCandidateRegionIdx`
   - 展示吸附高亮
6. 松手：
   - 有候选目标：交换两块顺序并落位
   - 无候选目标：回弹原位

### 8.3 空白区处理

- `swapping` 下拖到空白区域松手，不更新顺序，统一回弹。

## 9. 视觉反馈规范

1. `pressing`：无强反馈，仅保留轻微按下态（可选）。
2. `panning`：显示移动光标与当前格子激活态。
3. `swapping`：
   - 源块高层级显示
   - 放大 + 虚线 + 阴影
   - 候选目标块高亮 ring
4. `settling`：
   - 交换成功：短时吸附过渡动画
   - 交换失败：回弹过渡动画

## 10. 异常与边界处理

1. 任意 `pointerup/pointercancel/lostpointercapture` 均执行统一 cleanup。
2. 非当前 `pointerId` 输入全部忽略。
3. 候选目标仅允许非自身区域。
4. 候选目标距离相等时按最小索引稳定决策。
5. 模板切换或区域布局变化时，中断当前手势并重采样区域 rect。
6. 区域尺寸异常（0 宽高）时禁用平移更新，避免非法计算。

## 11. 验收标准

### 11.1 功能验收

1. 短按拖动能稳定进入平移，不需按空格。
2. 平移始终限制在当前格子边界内。
3. 长按 300ms 且位移 <= 4px 可进入换位模式。
4. 长按期间位移 > 4px 不进入换位，转为平移。
5. `swapping` 松手命中候选目标（<=24px）时完成交换。
6. 未命中候选目标时回弹且顺序不变。

### 11.2 体验验收

1. 模式转换无闪烁、无抖动。
2. 连续多次操作无状态卡死。
3. 右侧列表拖拽排序仍可正常使用。
4. 鼠标与触控行为一致。

## 12. 实施步骤建议

1. 引入手势状态与 Pointer 事件处理骨架。
2. 接入短按平移分支并复用边界计算。
3. 接入长按判定与换位分支。
4. 增加吸附命中与视觉反馈。
5. 移除画布区旧 `draggable` 换位路径。
6. 进行回归验证（平移、换位、回弹、列表拖拽）。

## 13. 风险与缓解

1. 风险：重构期间交互回归。
   - 缓解：按“平移先通、换位后接”的分步发布策略。
2. 风险：触控端行为不一致。
   - 缓解：全部基于 Pointer 事件，统一阈值与命中算法。
3. 风险：高频 move 造成渲染压力。
   - 缓解：必要时使用 `requestAnimationFrame` 节流视觉更新。

## 14. 兼容性说明

- 目标浏览器环境支持 Pointer Events。
- 若极端环境不支持 Pointer Events，可保留最小降级（仅短按平移，不启用长按换位）。
