# 照片右键菜单与待选区功能设计文档

## 1. 背景与目标

### 1.1 背景
当前周期详情页的照片选择交互方式为：
- 单击：切换选中状态（加入/移出候选）
- 双击：打开预览
- 选中后显示"设为最终"按钮

为了提供更丰富的操作方式和更好的用户体验，需要增加右键菜单功能，并新增待选区 tab。

### 1.2 目标
- 取消单击切换选中的交互方式
- 增加自定义右键菜单，提供更丰富的操作入口
- 新增待选区 tab，集中查看和管理当前周期的待选照片
- 保持双击预览功能不变

## 2. 需求概述

### 2.1 功能清单
1. 取消照片的单击切换选中事件
2. 新增自定义右键菜单，包含以下操作：
   - 加入到待选区 / 从待选区移除（切换选中状态）
   - 确认选择（设为最终照片）
   - 预览（打开预览弹窗）
3. 新增"待选区"tab 页面
4. 待选区 tab 显示当前周期的所有待选照片
5. 待选区中的照片也支持右键菜单操作

### 2.2 范围
- 仅针对照片，视频暂不支持
- 仅在周期详情页（PeriodSelectPage）生效
- 待选区按当前周期显示，随左侧周期切换而变化

## 3. 功能设计

### 3.1 Tab 结构调整

**原有结构：**
```
照片 | 视频
```

**新结构：**
```
照片 | 待选区 | 视频
```

- **照片 tab**：显示当前周期的所有照片
- **待选区 tab**：显示当前周期中 `is_selected = true` 的照片
- **视频 tab**：显示当前周期的所有视频（保持不变）

### 3.2 右键菜单功能

#### 3.2.1 菜单项
右键菜单根据照片状态动态显示：

| 菜单项 | 显示条件 | 功能说明 |
|--------|----------|----------|
| 加入到待选区 | `is_selected = false` | 将照片加入待选区，设置 `is_selected = true` |
| 从待选区移除 | `is_selected = true` | 将照片从待选区移除，设置 `is_selected = false` |
| 确认选择 | 始终显示 | 将照片设为最终照片，设置 `is_final = true` |
| 预览 | 始终显示 | 打开照片预览弹窗 |

#### 3.2.2 菜单特性
- 跟随鼠标右键点击位置显示
- 自动边界检测：超出视口时自动调整位置
- 点击页面其他地方自动关闭
- 按 ESC 键自动关闭
- 点击菜单项执行操作后自动关闭

### 3.3 待选区 Tab

#### 3.3.1 显示内容
- 只显示当前周期 `is_selected = true` 的照片
- 照片网格布局与"照片"tab 保持一致
- 显示最终照片标记（绿色对勾徽章）

#### 3.3.2 空状态
- 当没有待选照片时，显示空状态提示
- 提示文案："暂无待选照片"
- 辅助说明："在照片 tab 中右键点击照片，选择'加入到待选区'"

#### 3.3.3 交互
- 支持右键菜单（与照片 tab 相同）
- 支持双击预览
- 移除后照片从待选区消失
- 设为最终后，照片保留在待选区并显示最终标记

## 4. 交互设计

### 4.1 右键菜单触发
- 在照片上点击鼠标右键
- 阻止浏览器默认右键菜单
- 在鼠标点击位置显示自定义菜单

### 4.2 菜单定位
- 默认：菜单左上角对齐鼠标点击位置
- 右边界检测：如果菜单右边超出视口右边界，则菜单右对齐鼠标位置
- 下边界检测：如果菜单底部超出视口下边界，则菜单底部对齐鼠标位置

### 4.3 菜单关闭
- 点击任意菜单项：执行操作后关闭菜单
- 点击页面其他区域：关闭菜单
- 按 ESC 键：关闭菜单
- 滚动页面：关闭菜单（可选，视体验而定）

### 4.4 视觉样式
- 白色背景，圆角，阴影
- 菜单项 hover 时显示浅灰色背景
- 菜单项文字左对齐，图标在左（可选）
- 与现有 UI 风格保持一致

## 5. 技术实现方案

### 5.1 整体架构

```
PeriodSelectPage
├── Tab 导航（照片 | 待选区 | 视频）
├── 照片 tab
│   └── PhotoGrid（照片网格）
│       └── PhotoItem（照片项）
│           └── onContextMenu → 显示右键菜单
├── 待选区 tab
│   └── PhotoGrid（照片网格，过滤 is_selected = true）
│       └── PhotoItem（照片项，复用）
│           └── onContextMenu → 显示右键菜单
└── 视频 tab（保持不变）
```

### 5.2 组件设计

#### 5.2.1 PhotoContextMenu 组件
**位置：** `src/components/PhotoContextMenu.tsx`

**Props：**
```typescript
interface PhotoContextMenuProps {
  visible: boolean;
  position: { x: number; y: number };
  photo: Photo | null;
  onAddToPending: (photo: Photo) => void;
  onRemoveFromPending: (photo: Photo) => void;
  onSetFinal: (photo: Photo) => void;
  onPreview: (photo: Photo) => void;
  onClose: () => void;
}
```

**功能：**
- 根据照片状态动态渲染菜单项
- 处理菜单项点击事件
- 自动边界检测与位置调整

#### 5.2.2 PhotoItem 组件改造
- 移除 `onClick` 事件（取消单击切换选中）
- 保留 `onDoubleClick` 事件（双击预览）
- 新增 `onContextMenu` 事件（右键显示菜单）

### 5.3 状态管理

**现有状态复用：**
- `currentPhotos`：当前周期的所有照片（复用）
- `is_selected`：照片是否选中（复用，即待选状态）
- `is_final`：照片是否为最终（复用）

**新增状态：**
- `selectedTab`：当前选中的 tab，新增 `'pending'` 选项
- 右键菜单相关状态（位置、可见性、当前照片等）

### 5.4 数据流转

```
1. 用户右键点击照片
   ↓
2. 阻止默认事件，记录鼠标位置和当前照片
   ↓
3. 显示右键菜单
   ↓
4. 用户选择菜单项
   ├─ 加入/移出待选区 → 调用 togglePhotoSelect → 更新照片 is_selected 状态
   ├─ 确认选择 → 调用 setFinalPhoto → 更新照片 is_final 状态
   └─ 预览 → 调用 handleOpenPreview → 打开预览弹窗
   ↓
5. 关闭右键菜单
   ↓
6. 状态更新 → 自动刷新照片列表和待选区列表
```
