# 视频生成稳定性增强设计文档

## 1. 需求背景

当前视频生成功能存在以下稳定性问题：
- 错误处理不完善，FFmpeg 执行失败时没有详细的错误输出
- 生成过程无法取消或中断
- 生成失败后没有清理临时文件
- 导出记录状态管理不完整

本次设计目标是增强视频生成的稳定性，包括：错误处理、取消机制、资源清理和状态管理。

## 2. 设计方案

### 2.1 取消机制

使用 `Arc<AtomicBool>` 作为取消标志，在生成流程的关键步骤检查取消状态。

**数据结构**：
```rust
lazy_static::lazy_static! {
    static ref CANCEL_MAP: Mutex<HashMap<String, Arc<AtomicBool>>> = Mutex::new(HashMap::new());
}
```

**工作流程**：
1. 生成开始时，创建 `Arc<AtomicBool>` 并注册到 `CANCEL_MAP`
2. 在每个阶段（准备、AI 生成、FFmpeg 编码）开始前检查取消状态
3. 收到取消请求时，设置标志为 true
4. 生成结束（成功/失败/取消）时，从 `CANCEL_MAP` 中移除

### 2.2 错误处理增强

**FFmpeg 错误捕获**：
- 执行 FFmpeg 时保留 stderr 输出
- 将 stderr 内容作为错误信息的一部分

**导出记录状态更新**：
- 生成开始：`status = "processing"`
- 生成成功：`status = "success"`，更新 `file_size` 和 `duration`
- 生成失败：`status = "failed"`，更新 `error_message`

### 2.3 资源清理机制

**临时文件追踪**：
- 在生成过程中记录创建的临时文件路径（AI 帧、文字处理后的图片等）
- 生成失败或取消时清理这些临时文件

**清理时机**：
- 生成失败时立即清理
- 用户取消时立即清理
- 生成成功后清理 Agnes 临时目录

## 3. API 设计

### 3.1 新增 Tauri 命令

#### `cancel_generation`
- **用途**：取消正在进行的视频生成任务
- **参数**：`task_id: String` - 任务标识
- **返回**：`Result<(), String>`

### 3.2 修改现有命令

#### `generate_growth_video`
- **修改内容**：
  - 添加 `task_id` 参数
  - 支持取消检查
  - 增强错误处理

## 4. 前端改动

### 4.1 UI 改动

**取消按钮**：
- 在生成过程中显示取消按钮
- 点击后调用 `cancel_generation` 命令
- 取消后显示取消提示

**错误提示改进**：
- 显示更详细的错误信息
- 添加错误日志记录

## 5. 数据模型变更

### 5.1 导出记录扩展

`ExportRecord` 模型已包含所需字段：
- `status`: `success` | `failed` | `processing`
- `error_message`: Option<String>

无需新增字段。

## 6. 关键实现细节

### 6.1 取消检查点位置

**标准模式 (`generate_growth_video_async`)**：
1. 读取照片后
2. AI 过渡帧生成前（如果启用）
3. FFmpeg 编码前

**Agnes 模式 (`generate_growth_video_agnes`)**：
1. 读取照片后
2. 文字预处理前
3. Agnes API 调用前
4. 回退到标准模式前

### 6.2 错误传播路径

```
FFmpeg 执行失败
    ↓
捕获 stderr 输出
    ↓
构建详细错误信息
    ↓
更新导出记录状态为 failed
    ↓
向上传播错误
    ↓
前端显示错误提示
```

### 6.3 资源清理范围

**标准模式**：
- AI 帧目录（如果生成了 AI 帧）

**Agnes 模式**：
- Agnes 临时目录（`agnes_temp`）
- AI 帧目录（如果生成了 AI 帧）

## 7. 测试计划

### 7.1 功能测试

1. **正常生成测试**：验证正常流程的正确性
2. **取消测试**：验证取消功能是否正常工作
3. **错误测试**：验证错误处理是否正确
4. **资源清理测试**：验证临时文件是否被正确清理

### 7.2 边界测试

1. **空照片测试**：验证无照片时的错误提示
2. **无效路径测试**：验证输出路径无效时的错误处理
3. **取消时机测试**：验证在不同阶段取消的行为

## 8. 风险评估

| 风险点 | 影响 | 缓解措施 |
|--------|------|----------|
| 取消机制可能在步骤中间生效 | 用户体验略有影响 | 在关键步骤开始前检查取消状态 |
| 资源清理可能失败 | 磁盘空间占用 | 使用 `std::fs::remove_file` 的 `ok()` 忽略错误 |
| FFmpeg stderr 输出过大 | 内存占用 | 限制读取的 stderr 大小 |

## 9. 后续扩展

本设计为后续扩展预留了接口：
- 如果需要更复杂的取消机制，可以替换为 channel 方案
- 如果需要支持多任务管理，可以扩展 `CANCEL_MAP` 为更完整的任务注册表
- 如果需要支持进度保存和恢复，可以在此基础上添加状态持久化
