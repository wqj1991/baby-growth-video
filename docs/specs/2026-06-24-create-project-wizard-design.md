# 创建项目向导设计文档

**日期**: 2026-06-24
**状态**: 待审核
**版本**: 1.0

## 概述

为宝宝成长视频制作工具添加完整的创建项目向导流程，通过分步引导的方式帮助用户快速创建新的成长视频项目，提升用户体验。

## 背景与目标

### 背景
当前项目虽然有"新建项目"的入口，但没有完整的创建流程，用户体验不连贯。需要一个完整的向导来引导用户完成从选择宝宝到生成周期的全过程。

### 目标
- 提供清晰的 5 步创建流程
- 左侧步骤列表导航，直观展示进度
- 支持照片文件夹扫描和自动周期生成
- 创建完成后展示项目概览
- 状态管理清晰，支持步骤间往返修改

## 整体架构


## 向导页面设计

### 整体布局
- 左侧：步骤导航栏（固定宽度 200px）
- 右侧：步骤内容区域（自适应宽度）
- 底部：操作按钮栏（上一步 / 下一步 / 完成）
- 顶部：返回按钮 + 标题 + 取消按钮

### 左侧步骤栏（WizardSidebar）
- 显示所有 5 个步骤
- 状态样式：
  - **当前步骤**：蓝色背景 + 白色数字 + 文字加粗
  - **已完成步骤**：绿色对勾图标 + 灰色文字
  - **未到达步骤**：灰色数字 + 浅灰色文字
- 交互：点击已完成步骤可跳回修改

### 底部操作栏
- **上一步**：返回上一步骤（第一步时隐藏）
- **下一步**：进入下一步骤（当前步骤验证通过后启用）
- **完成**：最后一步显示，提交创建项目
- **取消**：右上角，放弃创建并确认后返回首页

## 各步骤详细设计

### 步骤 1：选择宝宝

**功能**：选择为哪个宝宝创建视频项目

**内容**：
- 宝宝列表卡片展示（头像、姓名、出生日期、性别）
- 单选模式，选中高亮
- 只有一个宝宝时默认选中
- 底部"添加新宝宝"按钮（跳转到宝宝设置页）

**验证规则**：
- 必须选中一个宝宝才能进入下一步

**边界情况**：
- 无宝宝时：显示空状态 + "添加宝宝"按钮

---

### 步骤 2：项目信息

**功能**：设置项目基本信息

**表单字段**：
1. **项目名称**（必填）
   - 输入框，默认值：`{宝宝名}成长视频`
   - 最大长度：50 字符

2. **项目描述**（选填）
   - 文本域，最多 200 字符
   - 占位符：简单介绍一下这个视频...

3. **周期天数**（必填）
   - 数字输入框，默认值：7
   - 取值范围：1-365 天
   - 快捷选项：7天（每周）、15天（半月）、30天（每月）、100天

4. **包含特殊日期**（可选）
   - 开关，默认关闭
   - 开启后自动添加：满月、百天、半岁、一岁 等特殊周期

**验证规则**：
- 项目名称必填
- 周期天数字段必填且在有效范围内

---

### 步骤 3：选择照片文件夹

**功能**：选择宝宝照片所在文件夹并扫描

**交互流程**：
1. 显示"选择照片文件夹"按钮
2. 点击后调用系统文件夹选择对话框
3. 选中后显示文件夹路径
4. 点击"开始扫描"执行扫描
5. 扫描过程显示进度条和状态文字
6. 扫描完成显示结果统计

**扫描结果展示**：
- 共扫描到 X 张照片、Y 个视频
- 成功识别日期：Z 张
- 无法识别日期：N 张
- 已自动去重：M 张

**验证规则**：
- 必须选择文件夹并完成扫描才能进入下一步
- 扫描到 0 张照片时警告但允许继续

---

### 步骤 4：生成周期

**功能**：根据出生日期和周期设置自动生成周期

**内容**：
- 信息摘要：宝宝出生日期、周期天数、是否包含特殊日期
- "生成周期"按钮
- 生成过程显示加载状态
- 生成完成后展示周期概览

**周期概览展示**：
- 共生成 X 个周期
- 时间范围：从出生到今天
- 周期列表（可折叠）：
  - 每个周期显示序号、时间范围、已分配照片数
  - 特殊周期（满月、百天等）高亮标注
- "重新生成"按钮（可调整参数后重新生成）

---

### 步骤 5：完成

**功能**：确认创建并展示成功状态

**内容**：
- 成功动画/图标
- 标题："项目创建成功！"
- 项目信息摘要卡片：
  - 宝宝姓名
  - 项目名称
  - 周期数量
  - 照片总数
- 操作按钮：
  - **查看项目概览**（主按钮）- 跳转到项目概览页
  - **立即开始选照片**（次按钮）- 直接跳转到周期选择页面

## 项目概览页设计

### 页面模块

#### 1. 基础信息卡片
- 宝宝头像、姓名、出生日期
- 项目名称、周期设置
- 创建时间

#### 2. 数据统计卡片（3个并排）
- **完成进度**：百分比 + 进度条
- **已选照片**：已选数 / 总周期数
- **剩余周期**：未选照片的周期数

#### 3. 快捷操作区
- **开始选照片**（主操作）- 跳转到周期选择页
- **生成视频** - 跳转到视频生成页
- **查看历史** - 跳转到历史记录页

#### 4. 周期概览网格
- 网格布局展示所有周期
- 每个周期显示：周期名称、状态（已选/待选）
- 已选周期显示照片缩略图
- 点击可直接跳转到对应周期

## 状态管理

### createProjectStore（Zustand）

```typescript
interface CreateProjectState {
  // 当前步骤
  currentStep: number;
  
  // 步骤1：宝宝
  selectedBabyId: number | null;
  
  // 步骤2：项目信息
  projectName: string;
  projectDescription: string;
  periodDays: number;
  includeSpecialDates: boolean;
  
  // 步骤3：文件夹
  folderPath: string | null;
  scanResult: ScanResult | null;
  isScanning: boolean;
  
  // 步骤4：周期
  periods: Period[];
  isGeneratingPeriods: boolean;
  
  // Actions
  setCurrentStep: (step: number) => void;
  setSelectedBaby: (babyId: number) => void;
  setProjectInfo: (info: ProjectInfo) => void;
  setFolderPath: (path: string) => void;
  setScanResult: (result: ScanResult) => void;
  setPeriods: (periods: Period[]) => void;
  reset: () => void;
}
```

## 数据流程

### 创建项目完整流程

```
1. 用户从首页点击"新建项目"
2. 进入 /create-project 路由
3. 初始化 createProjectStore
4. 步骤1：选择宝宝
5. 步骤2：填写项目信息
6. 步骤3：选择文件夹并扫描
7. 步骤4：生成周期预览
8. 步骤5：点击"创建项目"
   ├─ 调用 create_project 接口
   ├─ 调用 generate_periods 接口
   └─ 调用 scan_media_folder 接口
9. 创建成功，跳转到 /project/:id/overview
```

## 错误处理

### 扫描相关错误
| 场景 | 处理方式 |
|------|----------|
| 文件夹不存在/无权限 | 友好提示，重新选择 |
| 扫描到 0 张照片 | 警告提示，允许继续 |
| 部分照片识别失败 | 统计展示，不阻塞流程 |
| 扫描超时 | 可取消，支持重试 |

### 创建相关错误
| 场景 | 处理方式 |
|------|----------|
| 项目名称重复 | 表单验证提示 |
| 数据库错误 | 显示错误信息，保留数据可重试 |
| 网络异常 | 友好提示，重试按钮 |

### 边界情况
1. **无宝宝**：向导第一步显示空状态，引导添加宝宝
2. **中途退出**：点击取消时确认对话框，确认后丢弃数据返回
3. **浏览器刷新**：状态丢失，重新开始（第一版简化）

## 与现有代码集成

- 宝宝卡片样式（复用首页宝宝列表）

## 技术实现要点

1. **状态管理**：使用 Zustand 独立 store 管理向导状态，避免污染全局 store
2. **表单验证**：每步独立验证，通过后才能进入下一步
3. **异步操作**：扫描、生成周期等耗时操作显示 loading 状态
4. **路由守卫**：直接访问后续步骤时自动跳回第一步
5. **响应式布局**：左侧步骤栏在小屏幕下可改为顶部进度条
