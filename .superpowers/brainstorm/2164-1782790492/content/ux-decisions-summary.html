🔄 照片状态流转详解

每张 Photo 实体包含 3 个布尔标记，各自独立控制不同 UI 行为

3 个布尔标记
标记 含义 触发操作 清除条件
is_selected 已加入待选区 点击「加入待选区」 在待选区点击「移除」
is_multi_selected 多选打勾（拼图用） 在待选区勾选 checkbox 取消勾选或离开待选区
is_final 该周期最终选定 在待选区点击「单独选定」 点击「取消最终」

📌 5 条操作路径

路径 1 照片网格 ↔ 待选区 — 点击「+ 加入待选区」→ is_selected = true → addToSelectedItems()
↩ 回退A PhotoCard hover「✕ 从待选区取消」→ onToggleSelect → handleTogglePhotoSelect
↩ 回退B PendingPanel「移除」→ handleRemoveFromStash → handleTogglePhotoSelect
⚡ 通用效果 is_selected = false + is_multi_selected 强制重置 + removeFromSelectedItems() + pending_count -1
路径 2 待选区 → 拼图 — 勾选 ≥2 张 → 「生成拼图」→ TemplateSelector → CollageWorkspace
路径 3 待选区 ↔ 最终选定 — 点击「单独选定」→ is_final = true + Period.selected_photo_id → 周期状态 🟢
→ 入口A PhotoCard hover「✓ 设为最终」→ onSetFinal → handleSetFinalPhoto
→ 入口B PendingPanel hover「单独选定」→ onSelectSingle → handleSetFinalPhoto
↩ 回退 PhotoCard hover「✕ 取消最终」(红色) → onCancelFinal → handleCancelFinalPhoto()
⚡ 全局清除 所有 currentPhotos.is_final = false + 所有 selectedItems.is_final = false + selected_photo_id = undefined + has_final = false
路径 5 照片网格 → 最终选定 — PhotoCard hover「✓ 设为最终」→ handleSetFinalPhoto → is_final=true + selected_photo_id（跳过待选区）
↩ 回退 同一 PhotoCard hover「✕ 取消最终」→ 全局清除（同路径 3 回退）
⚡ 区别 照片未经过待选区，但 selectedItems 不受影响（照片不在其中）
路径 4 回退 — 「移除」退回网格 / 「取消最终」全局清除当前周期所有 is_final

🔑 5 个关键规则

待选区是主要操作入口 — 拼图、单独选定等核心操作在待选区执行。照片网格可直接「设为最终」（路径5，跳过待选区）
is_final 与 is_selected 共存 — 设为最终后照片仍留在待选区，可同时勾选 is_multi_selected 参与拼图
取消最终 → 全局回退 — handleCancelFinalPhoto 会清除当前周期所有照片的 is_final 和 selected_photo_id
周期级别的最终选定 — Period.selected_photo_id 指向最终选定的照片/视频帧 ID，是周期时间线的数据源
跨周期隔离 — 切换周期时，待选区 (selectedItems) 会根据新周期的 is_selected 照片重新加载

📡 数据流

照片选择操作 → Zustand Store (useAppStore)
├── selectedItems[] ← 待选区面板的数据源
├── currentPhotos[] ← 照片网格的数据源
├── periods[].selected_photo_id ← 周期时间线的数据源
└── periodStats[].has_final / pending_count ← 周期统计徽章的数据源
